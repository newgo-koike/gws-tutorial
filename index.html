第1章 Google Workspaceの全体像

まずは全体を把握しよう

1.1 Google Workspaceとは？

Google Workspace（旧 G Suite）は、Googleが提供するビジネス向けクラウドツールのセットです。メール、ファイル保存、ドキュメント作成、ビデオ会議など、仕事に必要なツールがすべて揃っています。

すべてのデータはクラウド（Googleのサーバー）に保存されるので、どのデバイスからでもアクセスでき、自動保存されるのでデータを失う心配がありません。

💡 クラウドとは？
自分のPCではなく、インターネット上のサーバーにデータを保存する仕組みです。スマホでもPCでも同じデータにアクセスでき、PCが壊れてもデータは安全です。

1.4 セキュリティ設定（最初にやるべきこと）

1

2段階認証を有効にする

Googleアカウント → セキュリティ → 2段階認証プロセス から設定。パスワードが漏れても、スマホの確認なしではログインできなくなります。

2

復旧用メール・電話番号を設定

パスワードを忘れた時の復旧手段。アカウント → 個人情報 → 連絡先情報で設定。

3

セキュリティ診断を実行

myaccount.google.com/security-checkup にアクセス。不審なログインやアプリ権限をチェック。

⚠️ 2段階認証は必須！
Googleアカウントにはメール、ファイル、カレンダーなど全ての重要データが入っています。2段階認証なしは「家の鍵をかけずに外出する」のと同じです。

2.2 ラベルとフィルタで自動整理

Gmailにはフォルダがありません。代わりに「ラベル」を使います。1つのメールに複数のラベルを付けられるのがフォルダとの違いです。

❌ フォルダ方式（他のメールソフト）

1つのメールは1つのフォルダにしか入れられない。「仕事」と「重要」の両方に入れたい時に困る。

✅ ラベル方式（Gmail）

1つのメールに「仕事」「重要」「プロジェクトA」など複数のラベルを付けられる。

フィルタの設定方法

条件に合うメールに自動でラベルを付けたり、アーカイブしたりできます。

1

検索バーの右端にある「▼（検索オプション）」をクリック

2

条件を入力（From、件名、キーワードなど）

3

「フィルタを作成」をクリック → アクションを選択

例：「ラベルを付ける: 仕事」「受信トレイをスキップ（アーカイブする）」「スターを付ける」など

✅ おすすめフィルタ例
ニュースレター・メルマガラベル「メルマガ」＋ 受信トレイスキップ
上司からのメールラベル「重要」＋ スター付き
特定プロジェクトの通知件名に「[ProjectA]」→ ラベル「ProjectA」

2.4 便利な設定と機能

✍️ 署名の設定

設定（⚙️）→ すべての設定を表示 → 全般 → 署名

新規作成と返信で別の署名を使い分けることもできます。HTML形式で画像やリンクも挿入可能。

📤 送信取り消し

設定 → 全般 → 送信取り消し で5〜30秒の猶予時間を設定。送信直後に「取り消し」ボタンが表示されます。30秒に設定するのがおすすめ。

📧 エイリアス（+記法）

メールアドレスの@の前に+任意の文字を追加しても同じアドレスに届きます。

例: koike+shopping@gmail.com → koike@gmail.com に届く

活用法：サービスごとに別のエイリアスで登録すると、フィルタで自動整理できます。

🕒 スヌーズ機能

「今は対応できないけど忘れたくない」メールを、指定した日時に受信トレイの先頭に再表示させます。メール上で右クリック → スヌーズ。

📅 送信日時の指定

送信ボタンの横にある「▼」→「送信日時を設定」で、未来の日時にメールを予約送信できます。深夜に書いたメールを翌朝に送りたい時に便利。

3.2 共有設定（超重要）

Google Driveの共有は3段階の権限と2つの共有方法の組み合わせです。

権限レベルできること適する場面
閲覧者見るだけ。編集・コメント不可資料の配布
閲覧者（コメント可）見る＋コメントできる。編集は不可フィードバック収集
編集者何でもできる（編集・共有設定の変更も）共同編集メンバー

👤 ユーザー指定で共有

メールアドレスを入力して特定の人に共有。最も安全な方法。

🔗 リンク共有

URLを知っている人なら誰でもアクセス可能。手軽だが、リンクが漏れると誰でも見られるので注意。

⚠️ リンク共有の危険性
「リンクを知っている全員が編集可」にすると、URLが流出した時にデータが書き換えられる可能性があります。社外への共有は「閲覧者」権限でが原則です。

関数意味使用例

💡 VLOOKUP vs XLOOKUP

VLOOKUP は左端列しか検索できず、列番号を数字で指定する必要があります。XLOOKUP はどの列でも検索でき、列追加にも強い。新しく書くなら XLOOKUP がおすすめです。

5.1 Google Apps Script（GAS）で自動化

Google Apps Script はGoogleが提供するプログラミング環境です。JavaScriptベースで、Gmail・Drive・スプレッドシートなどを自動操作できます。

💡 GASの始め方

スプレッドシートを開く → 拡張機能 → Apps Script → エディタが開く。ここにコードを書いて実行するだけ！

例1: スプレッドシートからメールを自動送信

// スプレッドシートの名前・メールアドレスを元に一括メール送信
function sendEmails() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const data = sheet.getDataRange().getValues();

  // 1行目はヘッダーなのでスキップ（i=1から開始）
  for (let i = 1; i < data.length; i++) {
    const name = data[i][0];     // A列: 名前
    const email = data[i][1];    // B列: メールアドレス
    const subject = "お知らせ";
    const body = name + "さん\n\nご連絡です。";

    GmailApp.sendEmail(email, subject, body);
  }
}

例2: 毎朝の日報リマインダー

// トリガーで毎日17:00に実行するよう設定
function sendDailyReminder() {
  const email = "team@example.com";
  const subject = "【リマインダー】日報の提出";
  const body = "本日の日報を提出してください。\nhttps://forms.google.com/xxx";

  GmailApp.sendEmail(email, subject, body);
}

例3: フォーム回答をSlackに通知

// フォーム送信時にSlackに通知を送る
function onFormSubmit(e) {
  const responses = e.response.getItemResponses();
  let message = "📣 新しいフォーム回答:\n";

  responses.forEach(r => {
    message += r.getItem().getTitle() + ": " + r.getResponse() + "\n";
  });

  const webhook = "https://hooks.slack.com/services/xxx";
  UrlFetchApp.fetch(webhook, {
    method: "post",
    contentType: "application/json",
    payload: JSON.stringify({ text: message })
  });
}
